项目心智地图：lift-app（电梯智能终端）

Java 11 Android 14+ RK3568 竖屏 167 个 Java 源文件 · 生成于 2026-06-18

一句话总结： 这是一个运行在 RK3568 电梯触摸屏上的 Android 梯媒终端 App， 核心机制是：通过 RS232 与按键板通信控制楼层/门，同时从云端拉取配置与广告排期，在竖屏 UI 上展示楼层按钮、广告、通知、欢迎语，并周期性上报心跳与播放日志。

🔀 数据流主路径

触控选层

用户点击楼层按钮 → MainActivity.onFloorTouch → ElevatorController.registerFloorTap (按下+松开帧对) → SerialManager 写 RS232 → 按键板响应 0x06 到站帧 → ElevatorController.onFrame 更新 ElevatorState → UI LiveData 刷新方向/楼层/注册灯

云端配置同步

板卡 SN 就绪 → TerminalPrefs 写入 terminalCode → RuntimeScheduler.bootstrap → HeartbeatWorker (WorkManager 链式 OneTimeWork) → LiftCloudClient.heartbeat (比对 revision) → pullConfig 拉取多域配置 (floor/skin/schedule/notice/...) → ConfigPullCoordinator → 各 Bridge 通知 UI/Engine 热更新 → AssetSyncWorker 下载皮肤/广告/语音文件

广告播放

ScheduleEngine 按 bizDate + 小时槽计算 PlayItem → AdPlaybackController 播放图片/视频 (Media3 ExoPlayer) → PlayLogger 记录播放事件入队 → PlayLogBatchWorker 批量上报云端

⚠️ 风险区域

🤖 AI / Vibe Coding 特征

  • ElevatorApplication 超过 600 行，承担过多装配职责（God Object 倾向）
  • 大量 ConfigPull*Bridge 类模式重复（Listener 桥接，可接受但维护成本高）
  • 约 25+ 处宽泛 catch (Exception)，部分会吞掉根因（如 YsBoardApi 7 处）
  • 根 README.md 仍写「脚手架待实现」，与代码实际完成度不一致（文档滞后）
  • config/env.dev.properties 提交了 bootKey 明文（应仅放 local.properties）

🔧 环境依赖

  • 构建：./gradlew :app:assembleDevDebug（flavor: dev / uat / prod）
  • 配置：config/env.<env>.properties + local.properties 覆盖
  • 必填：cloudBaseUrl；开发环境还需 bootKey（换 deviceSecret）
  • 硬件：RK3568 + 亿晟 ysapi.jar；RS232 默认 TTYS4
  • 外部服务：梯媒 Lift HTTP API、天气 API、新闻 API
  • 主要依赖：OkHttp、Gson、WorkManager、Media3 ExoPlayer、AndroidX Lifecycle
